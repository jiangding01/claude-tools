# Templates

Use these reusable templates when they fit the situation.

## First response

```markdown
### 🎯 核心命题对齐

我理解这个问题的核心不是「<表层问题>」，而是「<更本质的问题>」。

---

### 🔥 第 1 轮关键追问

问题：<只问一个最关键问题>

为什么重要：<说明它为什么影响后续判断>

我的当前判断：<你的倾向、假设或风险判断>
```

## Follow-up response

```markdown
### ✅ 当前对齐

你的回答已经明确了 <已明确内容>，但同时暴露出 <新的模糊点 / 假设 / 风险 / 分歧>。

---

### 🔥 下一轮追问

问题：<只问一个更深的问题>

为什么重要：<说明它影响什么>

我的当前判断：<你的倾向或风险判断>
```

```markdown
## 思维复盘与行动共识画布

### 1. 核心共识

<一句话总结最终对齐的结论。>

### 2. 被补齐的关键盲点

- <盲点 1>
- <盲点 2>
- <盲点 3>

### 3. 已确认的关键决策

- <决策 1>
- <决策 2>
- <决策 3>

### 4. 仍需关注的风险

- <风险 1>
- <风险 2>

### 5. 非目标 / 暂不处理事项

- <非目标 1>
- <非目标 2>

### 6. 下一步行动

- <角色 / Owner>：<行动项>
- <角色 / Owner>：<行动项>

### 7. 后续验证问题

- <问题 1>
- <问题 2>
```

## Advice mode

Use when the user explicitly asks for a direct recommendation.

```markdown
## 我的判断

<直接给出你的判断。>

## 判断依据

- <依据 1>
- <依据 2>
- <依据 3>

## 关键风险

- <风险 1>
- <风险 2>

## 我仍然需要确认的关键问题

问题：<只问一个会显著影响结论的问题>
```

## Fast review mode

Use when the user asks for a quick scan, not a full Socratic loop.

```markdown
## 快速审查：当前最关键的 3 个问题

1. <问题 1>
   - 为什么重要：<原因>

2. <问题 2>
   - 为什么重要：<原因>

3. <问题 3>
   - 为什么重要：<原因>

如果继续深入，我建议先从第 <n> 个问题开始追问。
```

## User-stuck helper

Use when the user cannot answer the question.

```markdown
你可以从这几个方向思考：

- 目标：<提示>
- 风险：<提示>
- 成本：<提示>

我建议你先回答其中最关键的一点：<单一问题>
```
