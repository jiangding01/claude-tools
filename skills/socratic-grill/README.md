## 它解决什么问题

事情通常是这样的：方案写完了，你觉得"基本没问题"，评审会上被连环追问三个回合，当场发现分片键没想清楚、灰度策略是拍的、收益口径根本没对齐。

直接问 AI"帮我看看这个方案"也不行——它会客气地夸两句，然后一口气甩给你十个问题外加一份你没要的完整解决方案。问题在于：**想清楚一件事需要的不是答案，而是被问对问题。**

这个 skill 把 AI 变成职场评审陪练：一次只问一个问题，问在最不可逆、最影响判断的点上；把你的"我们认为""问题不大"降级为待验证假设；用 PM/Dev/QA/SRE/Data/Manager 等 8 个角色的视角轮流压测；最后把对齐的结论烤成一张收敛画布。

## 效果示例

以下输出是真实评测产物（2026-06-09，claude-sonnet-4-6 实跑，未润色）：

**你说：**

> 功能开发完了，明天上线，灰度策略是直接全量，我感觉问题不大，就是改了个文案和一个小交互。

**它不问"回滚方案是什么"，而是先烤那个量词：**

> **问题：「问题不大」和「小交互」这两个判断，是基于代码改动量的认知，还是基于用户行为影响面的认知？**
>
> 为什么重要：如果是前者（改的代码少），那全量直接上线的风险敞口可能被低估了——文案和交互改动有时触达所有用户的核心操作路径，改动量小不等于影响面小。

如果「小」这个判断本身是错的，回滚、监控、止损全都会按错误的爆炸半径来设计。完整对话见 [examples/](examples/)。

## 快速开始

```bash
npx skills add jiangding01/claude-tools --skill socratic-grill
```

手动安装：

```bash
git clone https://github.com/jiangding01/claude-tools.git
cp -r claude-tools/skills/socratic-grill ~/.claude/skills/
```

装完对 Agent 说：

```text
我们打算把 MySQL 单库换成分库分表，主要为了解决性能问题，下周架构评审，我觉得方案基本没问题了。帮我烤一烤。
```

## 触发方式

说这些话都会触发：

- 帮我想清楚 / 帮我理清思路 / 我在纠结
- 这个方案有没有问题 / 帮我找找漏洞
- 帮我审一下这个 PRD / 技术方案
- 追问我 / 苏格拉底式提问
- 帮我复盘 / 帮我决策
- 前后端吵了两周没结论，帮我对齐一下分歧

**不会触发**：写代码、修具体 bug、解释概念、翻译、写邮件——你要的是交付物而不是想清楚时，它不出现。

## 它会交付什么

| 阶段 | 交付物 |
|---|---|
| 每一轮 | 一个问题 + 为什么重要 + 我的当前判断（不是问卷轰炸） |
| 假设积累 ≥3 个 | 开放假设清单，让你选最致命的先验证 |
| 发现分歧 | 分歧定位（10 种类型学：目标/范围/风险/证据/责任…） |
| 对话收敛 | 思维复盘与行动共识画布（共识/盲点/决策/风险/非目标/下一步） |

## 它和同类有什么不同

| 维度 | 常见苏格拉底类 skill | socratic-grill |
|---|---|---|
| 定位 | 教学/学习（引导你理解知识） | 职场评审陪练（帮你把方案想清楚） |
| 提问节奏 | 一轮 3–5 问的问卷波 | 严格单轮单问，问在最不可逆的点上 |
| 视角 | 单一导师视角 | 13 场景路由 × 8 角色透镜交叉压测 |
| 分歧处理 | 不覆盖 | 10 种分歧类型学 + 升级路径 |
| 产物 | 对话本身 | 收敛画布（可直接贴进会议纪要） |
| 效果证明 | 无 | [9 场景实测 98%，逐条可对账](evals/RESULTS.md) |

## 安全边界

- **不替你做决策**——它给判断和依据，决定权永远在你。
- **你喊停就停**——说"直接给建议"会切到建议模式，不会继续追问。
- **收敛画布先问后写**——主动提议，等你确认才生成。
- **零 API、零依赖、零外发**——纯提示词资产，不调用任何外部服务，不读取你的私有数据。

## 验证与测试

把 [evals/evals.json](evals/evals.json) 里任意一条 prompt 丢给装好 skill 的 Agent，合格表现是：只问一个问题、不重复你已说的信息、解释为什么这个问题重要、不直接附和你的"没问题"。

跑过 benchmark 后用脚本汇总：

```bash
python3 scripts/report_evals.py <workspace>/iteration-N/benchmark.json
```
